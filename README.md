# PRO-CHISTKA — управленческий дашборд

Дашборд для управления клининговым бизнесом: заявки, заказы, выручка, прибыль,
конверсия по каналам привлечения и экономика по видам услуг.

## Как пользоваться

Приложение — один файл `index.html` без зависимостей и сервера:

- **Локально** — скачайте `index.html` и откройте его в любом браузере.
- **Онлайн** — через GitHub Pages: в настройках репозитория
  (Settings → Pages) выберите «Deploy from a branch» и нужную ветку,
  папка `/ (root)`. Сайт будет доступен по адресу
  `https://dimpershko-a11y.github.io/prochistka_bisness/`.

## Что умеет

- **Учёт заказов**: заявка → в работе → выполнен/отменён; клиент, канал,
  услуга, площадь, сумма, прямые затраты, нормо-часы, оплата, заметки.
- **KPI за период**: выручка, валовая прибыль, маржа, конверсия
  заявка → заказ, средний чек, прибыль на нормо-час, долги клиентов,
  проблемные заказы.
- **Графики**: выручка и прибыль по месяцам, выручка по каналам и по услугам.
- **Таблицы-разрезы** по каналам привлечения и услугам — как в исходных
  Google-таблицах.
- **Фильтры**: период (всё время / месяц / прошлый месяц / 90 дней / год /
  свой диапазон), канал, услуга — фильтруют все цифры на странице сразу.
- **Тёмная и светлая тема**.

## Данные

Все данные хранятся локально в браузере (localStorage) — ничего никуда не
отправляется.

- **Экспорт** — резервная копия в JSON (восстанавливается через «Импорт»
  на любом устройстве).
- **CSV** — выгрузка всех заказов для Excel / Google Sheets.
- **Демо-данные** — кнопка на пустом дашборде, чтобы посмотреть, как всё
  выглядит с цифрами.

## Как считаются метрики

- **Заявки** — все записи, кроме отменённых, по дате заявки.
- **Заказы, выручка, прибыль** — только выполненные, по дате выполнения.
- **Валовая прибыль** = сумма заказа − прямые затраты (зарплата, химия, дорога).
- **Долг** = сумма заказа − оплачено (для выполненных заказов). Пустое поле
  «Оплачено» означает, что заказ оплачен полностью.

## Перенос данных из старой версии

Если в этом браузере остались данные предыдущей версии дашборда, при первом
открытии они переносятся автоматически: заказы — со статусами, затратами и
оплатой; несконвертированные заявки — как заявки. Старые данные при этом не
удаляются.
